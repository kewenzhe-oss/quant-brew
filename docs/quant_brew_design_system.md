# UI 设计语言提取与重构报告：从专业终端到 Quant-Brew

## A. 风格识别

- **关键词**：极客终端感 (Geek Terminal)、高信息密度、冷静克制、模块化工作台、实时监控感 (Live Trading Vibe)、严肃专业、数据纪律性。
- **成立逻辑总结**：这种风格之所以能够建立极强的专业信任感，是因为它摒弃了现代 Web 常见的“情绪留白”与“装饰圆角”，将屏幕的每一寸空间都压榨给了“数据”与“控制面板”。它通过极暗的纯黑底色搭配穿透力极强的高饱状态色，营造出一种“在暗房中操纵全球市场”的沉浸式权力感与专注力。它用等宽字体和1px线框建立了不容置疑的视觉秩序，让用户本能地相信这里的数据是精准、瞬时且冰冷的。

## B. 设计 DNA

### 1. 布局 DNA (Layout)
- **极窄全局栏**：顶部和底部通常被极窄的状态栏填充，包含微型的系统时钟、网络延迟（Live Dot）、服务器状态等。
- **硬切割网格**：主工作区摒弃页面滚动（No-scroll），采用全屏铺满的 Panel / Tile 拼图式布局，模块与模块之间通过 `1px` 锐利实线硬切割，无外部留白（Margin = 0）。

### 2. 配色 DNA (Color Logic)
- **黑阶底色（The Void）**：背景极其深沉（如 `#080808` 到 `#111111` 组成的三级灰阶），刻意避免带有任何温度的彩色灰。
- **极锐辅助色（Laser Accents）**：用来引导视线的主色往往非常锋利（如高亮橙、荧光青），且用量极克制，往往只作为激活 Tab 的底下划线（`2px`）或关键按钮的底色。
- **生硬状态色（Raw Status）**：涨跌红绿（`#16a34a` 绿 / `#dc2626` 红）不加过多修饰，甚至配合半透明同色背景作为 Badge 或 Highlight，形成强烈的“监控感”。

### 3. 字体 DNA (Typography)
- **等宽统治（Monospace Rule）**：核心数据、股票代号、报表标签强制使用等宽字体（如 Consolas / Courier），严格保证小数点上下绝对对齐（Tabular Lining）。
- **极小字号与全大写（Micro & Caps）**：大量分类标签和表头使用 `10px-11px` 的极小字号，配合 `font-weight: 700`、全大写（Uppercase）及 `letter-spacing: 0.5px-1px`（加宽字距），形成类似仪器仪表上的刻度文字感。

### 4. 组件 DNA (Components)
- **直角与微切角**：圆角极小（`radius: 2px` 或 `0`），没有任何柔软的现代感设定。
- **显性内部分割（Inner Borders）**：不靠留白区分内容，而是疯狂使用 `1px solid border-dim` 在卡片内部分割出 Header、Content、Footer。
- **紧凑型表格/列表**：极低的 cell padding（如 `2px 8px`），牺牲呼吸感以换取单屏能看 50 行数据的密度。

### 5. 交互 DNA (Interactions)
- **低延迟错觉**：没有弹簧动画、没有平滑过渡。Hover 态是生硬的亮度跳变（如从深灰跳到略浅的灰），传达出一种“无渲染延迟、立刻响应”的机器直觉。
- **长亮微光**：当前选中的节点或菜单项，会有很明确的边界长亮（Active State），像是物理控制台上的灯钮。

### 6. 信息密度 DNA (Density Strategy)
- **“降维重叠”策略**：在空间不足时，不用字号大小去拉开层级，而是通过亮度（Dim Text vs Primary Text）或者色彩（Muted 辅助色）在极高密度的文本块中制造视觉落差，让眼睛能自动过滤次要信息。

## C. 可迁移原则

将参考设计硬核的技术感，转化为“可用于新产品”的原创设计原则，服务于 **Quant-Brew** 的需求：

- **从“黑客暗房”迁移到“档案馆（Archive）”**：
  保留暗色基调的高级聚焦感，但把生硬的 `#080808` 替换为带有历史厚重感、Luxury Tech 色调的深墨灰（Ink）或老派终端绿灰，将“冰冷机器”升华为“价值提炼所”。
- **从“全盘监控”迁移到“叙事与焦点的平衡”**：
  保留 1px 硬切割的网格美学与高密度数据区，但引入“编辑精选感”。为“早晚报叙事缝合”区域加入大面积的优质排版空间；在“宏观经济”与“个股研究”区域则释放出金融终端的极致高密感。
- **从“仪表盘刻度字”迁移到“精致双字体系统”**：
  保留微型大写标签（Micro-Caps），但正文引入极具人文质感的粗细对比（例如：优美且克制的衬线体用于新闻叙事，冷峻的等宽字体用于行情点位与研报上下文）。

## D. 新产品重构方案：Quant-Brew

**定位**：一個極簡、奢華（Luxury Tech Archive）、由 AI 驅動的中文美股投研 Web 應用。
**核心支柱**：早晚報（敘事縫合）、宏觀經濟（定音錘）、個股研究（上下文過濾）。

### 1. 信息架构 & 页面框架
采用经典且严肃的 **Asymmetrical T-Layout（非对称T型布局）**：
- **全局环境（Top Bar 极度克制）**：去掉繁杂的菜单，仅保留时间戳（LOC/UTC）、大市冷暖（VIX/SPY 迷你 Ticker）、AI 状态灯（表示 AI 引擎正在运行过滤）。
- **左侧 / 中间：叙事与定音锤（60% 宽度）**
  - 早晚报叙事流：像高级报纸（如 FT）的数字版，内容被 AI “缝合”成流畅的阅读块。
  - 宏观经济悬浮板：嵌在叙事流的侧面或顶部，如同报纸上的数据角。
- **右侧：个股上下文与透视（40% 宽度）**
  - 侧边高密度栏：剥离无用信息，纯数据流。当左侧阅读到某公司时，右侧自动过滤并展示关联的估值水位、异常资金变动、AI 总结的关键因素。

### 2. 设计 Token & 视觉建议
**颜色体系 (Luxury Archive Theme)**：
- `bg-base`: `#1C1C1E`（一种带有深邃哑光质感的石墨黑，比纯黑更有温度）
- `bg-surface`: `#242426`（用于卡片和模块底色）
- `border-dim`: `#3A3A3C`（低调的线框，用于构建网格秩序）
- `text-primary`: `#F5F5F7`（不刺眼的珍珠白）
- `text-secondary` / `text-caption`: `#8E8E93`（适合于说明性的微型文本）
- `accent-gold` (Brand/Action): `#D4AF37` / `#B8860B`（褪色的旧金色，代表“brew/提炼”的隐喻，替代原版的刺眼亮橙色）
- `status-live`: `#00FA9A` (高荧光感的中春绿，点亮复古色板中的科技感)

**字体系统建议 (Typography)**：
- **大标题 / 叙事正文 (Serif/Classy)**: `Noto Serif SC` / `Songti SC` 配合 `Spectral` 或 `Playfair Display`。带来一份严肃的《华尔街日报》或旧档案的复古高级感。
- **数据面板 / 代码 / Ticker (Monospace)**: `JetBrains Mono` 或 `Geist Mono`。字形锐利，充满现代 Tech 质感。
- **微型标签 (Micro-UI)**: `Inter` 或 `San Francisco (SF Pro)`，字重600-700，全大写，字距加宽。

**核心比例 (Metrics)**：
- `Spacing`: 以 `4px` 为基准。内容区 padding 在 `12px` ~ `16px` 内收敛。
- `Radius`: `0px`。不要圆角，全部采用极其凌厉的直角切边，以凸显 Luxury Tech 的硬核感与确定性。
- `Border`: 大量使用 `1px solid` 分块，彻底摒弃 `box-shadow`（不需要空间投影，仅靠框线划分内容，扁平而专注）。

### 3. 组件清单
- **The Stitcher (叙事缝合块)**：带有衬线体标题和微型时间戳的阅读卡片，左侧带有 2px 的褪色金（Gold）竖向标线以表示 AI 生成。
- **Macro Hammer (定音锤 Ticker)**：包含宏观指数和货币的微型网格面板，数字采用亮白色，涨跌幅采用暗色加底色块的胶囊 `Badge`。
- **Context Filter panel (上下文透视面板)**：展示当前正在阅读公司的高密数据，由等宽字体整齐排列而成的 Table 列表，行高压缩到 24px。

## E. 防抄袭检查（避坑指南）

如果在重构新版本时仅仅修改颜色，依然会被看作是对原版金融终端（或类似彭博/FinceptTerminal）的拙劣模仿。以下是必须主动剥离的“原站残骸”：

1. **废弃复杂的顶部子系统选项卡**：原版顶部包含过多的模块（Maktets, Crypto, AI Chat, Backtesting 等）。*原创改写*：我们走极简路线，没有顶栏分类菜单。所有信息的呈现是“跟随视线而动态联动”的（读左边的新闻，右边自动过滤数据）。
2. **拒绝原版的“组件堆饰”**：原版在边框、Scrollbar 等地方使用了大量的 `1px` 辅助线条和复杂按钮（如 Detach/Dock 等原生行为按钮）。*原创改写*：Quant-Brew 作为 Web UI，抛弃所有窗体管理逻辑，让布局本身就化入石墨色的基底中，通过极简的 Typography 排版撑起设计感。
3. **改变品牌高光色的应用场景**：原版使用鲜艳的橙色/黄色代表专业。*原创改写*：不要原封不动照搬高饱和警惕色体系。引入「褪色的旧金色/香槟金」来作为强调色，搭配大量的优质「暖白衬线字体」以构建阅读体验的“低调奢华感”，这彻底脱离了旧系统“黑底亮字”的电子游戏感或单纯的黑客感。
